What is a Mapping in Solidity?

A mapping in Solidity is a key-value data structure, similar to a hash table or dictionary in other programming languages. It is commonly used to associate wallet addresses with balances on the blockchain.

Mappings are fundamental in Solidity, especially for storing and managing data such as user balances, permissions, or any other relationship between two types.

Basic Usage

The typical syntax for declaring a mapping is as follows:

mapping(_KeyType => _ValueType)

For example, to map wallet addresses to their balances:

mapping(address => uint) public balances;

What Can You Do with Mappings?

  • The key type can be almost any type (except for reference types like another mapping, a dynamically sized array, or a contract).
  • You can efficiently look up values (e.g., check a user’s balance by their address).

Limitations of Mappings

  • You cannot iterate over all keys in a mapping (no looping through keys).
  • You cannot retrieve a key by its value (no reverse lookup).

Mappings are designed for efficient storage and lookup, but they do not support enumeration or reverse searching. If you need to iterate over all elements, consider using an array or another data structure in combination with your mapping.
