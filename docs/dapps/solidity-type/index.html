Data Types in Solidity

Solidity is a statically-typed language, meaning that you need to specify the data type of a variable when you declare it. This allows the compiler to check for type errors and ensure that your code is safe and efficient.

Here are some of the most common data types in Solidity:

Integers

Integers are whole numbers, which can be either signed (positive or negative) or unsigned (positive only).

int b = -10; // Signed integer
uint a = 5;  // Unsigned integer

You can specify the size of an integer using keywords like int8, int16, int32, int64, uint8, uint16, uint32, and uint64. The default size for int and uint is 256 bits.

Booleans

Booleans represent truth values and can be either true or false.

bool bt = true;
bool bf = false;

Strings

Strings are sequences of characters enclosed in double quotes.

string hw = "Hello World";

Fixed-size Byte Arrays (bytesN)

Fixed-size byte arrays are used to store a fixed number of bytes. The size of the array is specified by the number N in the type declaration.

bytes32 eth = "ETH";

In this example, eth is a byte array that can store up to 32 bytes.

Address

The address type represents a 20-byte Ethereum address.

address adr = 0x06012c8cf97BEaD5deAe237070F9587f8E7A266d;

You can use the address type to store the addresses of contracts or external accounts.

For a complete list of data types and more detailed explanations, refer to the official Solidity documentation:

https://solidity-ja.readthedocs.io/ja/latest/types.html
